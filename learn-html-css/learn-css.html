Chapter 01

Introduction to CSS

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a document written in HTML or XML. If HTML is the skeleton of a website, CSS is the skin, makeup, and clothing.

Without CSS, websites would look like plain Word documents from the 90s. CSS allows you to control the layout, color, fonts, and responsiveness of a web page.

How CSS Works

When a browser displays a document, it must combine the document's content with its style information. It processes the document in two stages:

  1. The browser converts HTML and CSS into the DOM (Document Object Model).
  2. The browser displays the contents of the DOM.

Syntax Anatomy

A CSS rule-set consists of a selector and a declaration block.

style.css
/* Selector { Property: Value; } */
h1 {
  color: blue;
  font-size: 24px;
}
Chapter 02

Ways to Apply CSS

1. External CSS (Industry Standard)

This involves creating a separate file with a .css extension. This is the most efficient method because the browser caches the file, speeding up page load times on subsequent visits.

<!-- In HTML Head -->
<link rel="stylesheet" href="style.css">

2. Internal CSS

Internal styles are defined within the <style> element, inside the head section of an HTML page. This is often used for single-page websites.

3. Inline CSS

Inline styles are used to apply a unique style for a single element. Warning: This has the highest specificity and is hard to override later. Avoid using this in production.

<h1 style="color: blue; text-align: center;">Hello World</h1>
Chapter 03

CSS Selectors (Deep Dive)

Selectors are patterns used to select the element(s) you want to style. Understanding specificity is key to mastering CSS.

The Specificity Hierarchy

When multiple rules apply to the same element, the one with higher specificity wins. It is calculated as a 4-column value: (Inline, ID, Class, Tag).

  • Inline Style: (1, 0, 0, 0) - Most specific.
  • ID Selector (#header): (0, 1, 0, 0).
  • Class (.btn), Attribute, Pseudo-class: (0, 0, 1, 0).
  • Element (div), Pseudo-element: (0, 0, 0, 1).

Advanced Combinators

/* Descendant: Any p inside div */
div p { color: red; }

/* Child: Only direct children p */
div > p { color: blue; }

/* Adjacent Sibling: P immediately after div */
div + p { color: green; }

/* General Sibling: All P after div */
div ~ p { color: yellow; }
Chapter 04

Comments & Units

Absolute vs Relative Units

Absolute (px): Fixed units. 20px will always be 20 pixels regardless of screen size. Good for borders, bad for responsive text.

Relative (%, em, rem): These scale based on something else.

REM vs EM

REM (Root EM): Relative to the html font size (default 16px). If user changes browser font size, REM scales. Use this for font-sizes and margins.

EM: Relative to the parent element's font size. Good for padding inside buttons where padding should scale with the button text.

Viewport Units

  • 100vw = 100% of Viewport Width.
  • 100vh = 100% of Viewport Height.
  • vmin = Smaller of vw or vh.
Chapter 05

Colors & Backgrounds

Modern Color Formats

While Hex and RGB are common, modern CSS prefers HSL (Hue, Saturation, Lightness) because it is intuitive to adjust.

color: hsl(200, 100%, 50%); /* Base Blue */
color: hsl(200, 100%, 30%); /* Darker Blue */

Gradients

CSS allows linear and radial gradients.

.hero {
  background: linear-gradient(45deg, #ff0000, #0000ff);
}

Chapter 07

The Box Model (Vital)

The CSS Box Model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content.

[Image of Box Model]

Box Sizing Issue

By default, if you set width to 100px and padding to 20px, the actual width becomes 140px (100 + 20 left + 20 right). This breaks layouts.

The Universal Fix

Always use the following snippet at the top of your CSS file. It forces padding and border to be included inside the width.

*, *::before, *::after {
  box-sizing: border-box;
}
Chapter 08

Display & Visibility

The display property is the most important CSS property for controlling layout.

ValueBehavior
blockTakes full width, starts on new line (div, p, h1).
inlineTakes only content width, stays on same line (span, a). Margin-top/bottom doesn't work.
inline-blockStays on same line, but respects height/width/margins.
noneRemoves element from DOM flow completely.
Chapter 09

Dimensions & Overflow

Controlling size and scrolling behavior.

Max-Width

Using max-width: 100% ensures images never overflow their container on mobile screens.

Overflow

  • visible: Default. Content spills out.
  • hidden: Clips content. Useful for clearing floats.
  • scroll: Always shows scrollbar.
  • auto: Shows scrollbar only if needed.
Chapter 10

Positioning & Z-Index

The position property takes elements out of the normal document flow.

Types

  1. Static: Default.
  2. Relative: Positioned relative to itself. Unlocks z-index.
  3. Absolute: Positioned relative to the nearest positioned ancestor.
  4. Fixed: Sticks to the viewport (screen). Good for navbars.
  5. Sticky: Acts like relative until scrolled to a point, then becomes fixed.

Z-Index

Z-index only works on positioned elements (relative, absolute, fixed, sticky). Higher numbers go on top.

Chapter 11

Float & Clear

Historical Note: Floats were originally for wrapping text around images. Before Flexbox/Grid, they were used for layout hacks. Today, rely on Flex/Grid.

If you use float, you must use clear: both or a "clearfix" hack to prevent the parent container from collapsing height.

Chapter 12

Flexbox Guide

Flexbox is a layout module designed for laying out items in a single dimension (row OR column).

Container Properties

.flex-container {
  display: flex;
  flex-direction: row | column;
  justify-content: center | space-between; /* Main Axis */
  align-items: center | stretch; /* Cross Axis */
  flex-wrap: wrap;
  gap: 20px;
}

Item Properties

  • flex-grow: How much the item grows to fill space.
  • flex-shrink: How much it shrinks if space is tight.
  • flex-basis: Initial size before growing/shrinking.
Chapter 13

CSS Grid

Grid is the most powerful layout system in CSS. It handles both rows and columns simultaneously.

.grid-container {
  display: grid;
  /* 3 columns: 1st is 100px, 2nd is flexible, 3rd is 200px */
  grid-template-columns: 100px 1fr 200px; 
  gap: 20px;
}

Fraction Unit (fr)

The fr unit represents a fraction of the available space in the grid container.

Chapter 14

Responsive Design

Responsive Web Design (RWD) makes web pages render well on a variety of devices. We primarily use Media Queries.

Standard Breakpoints

/* Mobile First (Base styles are mobile) */
body { font-size: 14px; }

/* Tablet */
@media (min-width: 768px) {
  body { font-size: 16px; }
}

/* Desktop */
@media (min-width: 1024px) {
  body { font-size: 18px; }
}
Chapter 15

Transitions

CSS transitions allows you to change property values smoothly, over a given duration.

Required properties: transition-property (what to animate) and transition-duration (how long).

.btn {
  background: blue;
  transition: background 0.3s ease-in-out;
}
.btn:hover {
  background: green;
}
Chapter 16

Animations

Animations allow an element to change from one style to another. You can change as many CSS properties you want, as many times as you want.

@keyframes slideIn {
  0% { transform: translateX(-100%); opacity: 0; }
  100% { transform: translateX(0); opacity: 1; }
}

.box {
  animation: slideIn 1s ease forwards;
}
Chapter 17

Transforms

Transforms allow you to modify the coordinate space of the CSS visual formatting model. Using it, elements can be translated, rotated, scaled, and skewed.

  • translate(x, y): Moves element (Performance friendly).
  • scale(1.5): Zooms in/out.
  • rotate(45deg): Rotates element.
Chapter 18

Advanced Visuals

Box Shadow

Adds shadow effects around an element's frame.

box-shadow: 10px 10px 5px rgba(0,0,0,0.5); (x-offset, y-offset, blur, color).

Glassmorphism (Backdrop Filter)

.glass {
  background: rgba(255, 255, 255, 0.1);
  backdrop-filter: blur(10px);
  border: 1px solid rgba(255,255,255,0.2);
}
Chapter 19

CSS Variables

Also known as Custom Properties. They allow you to define a value once and reuse it throughout the document. This is critical for Dark Mode implementations.

:root {
  --primary-color: #3498db;
}

body {
  background-color: var(--primary-color);
}
Chapter 20

Modern Features

CSS3 is constantly evolving. Here are powerful new tools:

  • calc(): Perform math in CSS. width: calc(100% - 50px);
  • clamp(min, val, max): Responsive text sizes. font-size: clamp(1rem, 2.5vw, 2rem);
  • aspect-ratio: Define strict width-to-height ratio. aspect-ratio: 16 / 9;
  • scroll-behavior: smooth;: Enables smooth scrolling on anchors.
Chapter 21

Styling Forms

Default browser inputs look ugly. Use CSS to strip defaults and style them.

  • Use appearance: none; to remove default OS styling.
  • Style :focus states for accessibility (outline or box-shadow).
  • Use ::placeholder pseudo-element to style placeholder text.
Chapter 22

Styling Tables

Make data easy to read.

  • border-collapse: collapse; is essential to remove double borders.
  • Use tr:nth-child(even) for zebra striping rows.
Chapter 23

CSS Architecture (BEM)

Writing maintainable CSS is hard. BEM (Block Element Modifier) is a naming convention that helps.

  • Block: .card (Standalone entity)
  • Element: .card__title (Part of the block)
  • Modifier: .card--featured (Variation of the block)
Chapter 24

Performance

Speed Tips

  • Minify CSS files (remove whitespace) for production.
  • Avoid @import inside CSS files (blocks loading). Use <link> instead.
  • Avoid animating expensive properties like width, height, or top. Animate transform and opacity instead.
Chapter 25

Cross-Browser Compatibility

Not all browsers (Chrome, Safari, Firefox) support all features instantly. Use Vendor Prefixes to ensure support.

.box {
  -webkit-transform: rotate(45deg); /* Chrome/Safari */
  -ms-transform: rotate(45deg);     /* IE */
  transform: rotate(45deg);         /* Standard */
}
Chapter 26

CSS with JavaScript

You rarely write inline CSS with JS. Instead, use JS to toggle classes.

// Good Practice
element.classList.add('active');

// Bad Practice
element.style.backgroundColor = 'red';
Chapter 27

CSS Frameworks

Frameworks speed up development by providing pre-written CSS.

  • Bootstrap: Component-based (buttons, navbars). Good for beginners.
  • Tailwind CSS: Utility-first. You write classes like flex p-4 bg-red-500. Used by pros.
