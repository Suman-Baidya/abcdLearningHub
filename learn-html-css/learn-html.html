Chapter 01

Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language for creating Web pages. It describes the structure of a web page semantically and originally included cues for the appearance of the document.

HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects, such as interactive forms, may be embedded into the rendered page.

Key Concepts

  • HyperText: Refers to links that connect web pages to one another.
  • Markup: HTML uses "tags" to annotate text, images, and other content for display in a Web browser.
  • Language: It has a specific syntax and rules.

Chapter 02

Document Structure

An HTML document must follow a specific structure to be valid. The browser reads this structure from top to bottom.

boilerplate.html
<!DOCTYPE html> <!-- 1. Declaration -->
<html lang="en"> <!-- 2. Root Element -->
  
  <head> <!-- 3. Information for Browser -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
  </head>

  <body> <!-- 4. Visible Content -->
    <h1>Hello World</h1>
    <p>This is my first website.</p>
  </body>

</html>

Element Breakdown

  • <!DOCTYPE html>: Tells the browser "This is an HTML5 document".
  • <html>: The root element wrapping everything.
  • <head>: Contains meta-data, title, styles, scripts (not visible on page).
  • <body>: Contains text, images, videos (visible to user).

Chapter 04

HTML Attributes

Attributes provide additional information about elements. They are always specified in the start tag and usually come in name/value pairs.

Global Attributes (Work on any tag)

  • id="unique": Used to identify a single element (for CSS/JS).
  • class="group": Used to identify a group of elements.
  • style="...": Inline CSS styling (Avoid if possible).
  • title="text": Shows a tooltip on mouse hover.
  • hidden: Hides the element from view.
  • tabindex="0": Controls tab order for keyboard navigation.

Warning

IDs must be unique on a page. Classes can be reused on multiple elements.

Chapter 05

Links & Navigation

The <a> (Anchor) tag creates hyperlinks. The href attribute determines the destination.

<!-- Absolute URL (External Site) -->
<a href="https://google.com" target="_blank">Open Google</a>

<!-- Relative URL (Internal Page) -->
<a href="/about.html">About Us</a>

<!-- ID Link (Jump to section) -->
<a href="#contact">Go to Contact</a>

<!-- Email & Phone -->
<a href="mailto:test@test.com">Email Me</a>
<a href="tel:+1234567890">Call Me</a>

Pro Tip

Use target="_blank" to open links in a new tab, but for security, browsers often add rel="noopener noreferrer" automatically.

Chapter 06

Images & Media

Images

The <img> tag embeds an image. It is a void element (no closing tag). The alt attribute is required for screen readers and SEO.

<img src="images/logo.png" alt="Company Logo" width="300">

Figure & Figcaption

Use this for images with captions.

<figure>
  <img src="chart.jpg" alt="Sales Chart">
  <figcaption>Figure 1: Sales Growth 2025</figcaption>
</figure>

Video & Audio (HTML5)

Native support for media without Flash player.

<video controls autoplay muted loop poster="thumb.jpg">
  <source src="movie.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>
Chapter 07

Lists

1. Unordered List

Use <ul> for items where order doesn't matter (bullet points).

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JS</li>
</ul>

2. Ordered List

Use <ol> for numbered steps.

<ol>
  <li>Boil water</li>
  <li>Add Tea</li>
</ol>

3. Description List

Use <dl> for terms and definitions.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>
Chapter 08

Tables

Tables allow you to arrange data into rows and columns. They consist of <table>, <tr> (row), <th> (header), and <td> (data).

<table border="1">
  <thead>
    <tr>
      <th>Name</th>
      <th>Role</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>Developer</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>Designer</td>
    </tr>
  </tbody>
</table>

Important

Do NOT use tables for page layout. They are strictly for tabular data.

Chapter 09

Forms & Input (Crucial)

Forms are used to collect user input. The <form> element wraps various input controls.

Input Types

  • text: Single line text.
  • password: Masked characters.
  • email: Validates email addresses.
  • radio: Select ONE from a group.
  • checkbox: Select MULTIPLE options.
  • date: Date picker.
  • file: File upload.

Real World Example: Registration

<form action="/submit" method="POST">
  <fieldset>
    <legend>Personal Info</legend>
    
    <label>Username:</label>
    <input type="text" required placeholder="Enter name">
    
    <label>Skill:</label>
    <select name="skill">
      <option>HTML</option>
      <option>CSS</option>
    </select>
    
    <button type="submit">Register</button>
  </fieldset>
</form>

Interview Questions

Q: Difference between GET and POST?
A: GET sends data in the URL (visible, limited size). POST sends data in the HTTP body (hidden, secure, unlimited size).
Chapter 10

Semantic HTML

Semantic tags describe their meaning to both the browser and the developer. This is crucial for SEO and Accessibility.

  • <header>: Introductory content, logo, navigation.
  • <nav>: Set of navigation links.
  • <main>: The dominant content of the body (use once).
  • <section>: A thematic grouping of content.
  • <article>: Independent, self-contained content (Blog post, News).
  • <aside>: Content aside from the content (Sidebar).
  • <footer>: Footer for a document or section.
Chapter 11

Layout Techniques

HTML creates the structure. CSS creates the layout. However, how you structure HTML affects the layout.

Block vs. Inline

  • Block Elements: Start on a new line and take full width. (div, h1, p, section).
  • Inline Elements: Do not start on a new line, take only necessary width. (span, a, img, strong).

In modern web dev, we wrap content in Semantic Tags or divs and use CSS Grid/Flexbox to arrange them.

Chapter 12

Connecting Resources

Your HTML file is the central hub. You must link external files here.

<head>
  <!-- Link External CSS -->
  <link rel="stylesheet" href="style.css">
  
  <!-- Link External JavaScript -->
  <!-- 'defer' means load JS after HTML is parsed -->
  <script src="app.js" defer></script>
</head>

Iframe & Embeds

Chapter 15

Meta Tags & SEO

SEO (Search Engine Optimization) relies heavily on the <head> section.

Essential Meta Tags

  • charset="UTF-8": Ensures special characters display correctly.
  • viewport: Makes the site responsive on mobile phones.
  • description: The snippet shown in Google search results.
  • keywords: (Obsolete for Google, but used by some engines).
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="Learn HTML5 from scratch completely free.">
Chapter 16

Accessibility (A11Y)

Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them.

Checklist

  • Use alt text on all images.
  • Ensure forms have associated labels.
  • Use correct heading hierarchy (H1 -> H2 -> H3). Do not skip levels.
  • Ensure keyboard navigability using tabindex correctly.
Chapter 17

HTML APIs

HTML5 brought powerful APIs that interact with JavaScript to add advanced functionality.

1. Geolocation API

Allows the user to provide their location to the web application.

2. Drag and Drop API

Allows elements to be draggable.

3. Web Storage API

More secure and larger than cookies.

  • localStorage: Data persists even after the browser is closed.
  • sessionStorage: Data is lost when the browser tab is closed.
Chapter 18

Canvas & SVG

HTML5 allows you to draw graphics on the web page.

SVG (Scalable Vector Graphics)

XML-based. Great for logos and icons because they scale without losing quality.

<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" fill="red" />
</svg>

Canvas

JavaScript-based drawing. Great for games and complex animations.

<canvas id="myGame"></canvas>
Chapter 19

Performance

Writing clean HTML can speed up your website.

  • Lazy Loading: Use loading="lazy" on images. The browser only downloads them when the user scrolls near them.
  • Minification: Removing comments and white space from code before publishing.
  • Script Placement: Put scripts at the bottom of the body or use defer to prevent blocking rendering.
Chapter 20

Security Basics

HTML is just markup, but it's the entry point for many attacks.

Security Tips

  • Cross-Site Scripting (XSS): Never trust user input. Always sanitize data before displaying it.
  • Phishing via Links: When using target="_blank", always add rel="noopener noreferrer".
  • HTTPS: Always serve your website over HTTPS.

Project Ideas

Create a homepage about yourself.

  • Use an img for your photo.

2. Blog Page Layout

Practice semantics.
